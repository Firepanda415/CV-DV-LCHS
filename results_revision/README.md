# results_revision/ — status note

All paper-bound data and its provenance for the revised submission are
documented in **`../results_revision_v2/README.md`**. The only entries in this
directory that are still in active use are listed below; the remaining files
relate to the first submission and its pre-registration audit chain and are
**retained without further itemization**.

## Active entries

- `selected_params_v2.json` — current shared-operating-point selection record
  (the single parameter set (1.6, 0.25, 0.5, 32) used throughout the paper;
  selection rule, input hashes, supersession note).
- `viz_dense/landscape_dense.csv`, `viz_dense/landscape_dense_ext.csv`
  (+ grid JSON / meta / log) — the coarse-grid-search data behind both the
  parameter selection and the sensitivity figure (generated by
  `make_landscape_dense.py`; reproduces the registered subgrid 243/243
  bitwise).
- `gate_summary.json` — governance ledger (Gate A/B PASS; the B-amendment
  entry was added manually by the PI, with its full wording preserved in the
  entry itself).
- `convention.json`, `convention_coefficients.json` — Gate A convention
  evidence (inherited by v2).
- `coefficient_backend.csv` — coefficient-backend selection evidence
  (inherited by v2).
- `paper_hbar2/sensitivity_landscape_v2.meta.json`,
  `paper_hbar2/scaling_study.meta.json` — meta sidecars of the two current
  figures.

Caution: the four `*_table.csv` files under `paper_hbar2/` are old-point
exports that share names with the current ones — manuscript tables must be
filled from `../results_revision_v2/paper_hbar2/` only.
